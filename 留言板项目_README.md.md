# 留言板系统（基于Flask+Docker）

一款轻量、易用、功能完善的留言板系统，支持前台留言、多图上传、管理员后台管理、邮件通知、留言点赞/回复、前台显示控制等功能，部署简单，数据持久化，适配个人/小型站点使用。

## 📋 项目功能

### 前台功能

- 游客/用户可填写昵称、留言内容（支持Markdown格式）

- 多图上传：最多可上传4张图片（单张≤2M，支持PNG/JPG/GIF/SVG格式），支持预览、删除

- 表情插入：内置20个常用表情，方便快速编辑留言

- 留言展示：按时间倒序显示（最新留言在最上方），显示留言人、时间、IP归属地

- 留言点赞：支持对留言进行点赞，实时显示点赞数

- 管理员回复展示：显示管理员对留言的回复及回复时间

- 版权展示：底部显示版权LOGO（可配置）、版权文字及链接

### 后台管理功能

- 管理员登录/退出：密码加密存储，安全可靠

- 留言管理：查看所有留言（显示隐藏/显示状态）、删除留言、修改留言点赞数

- 留言回复：对任意留言添加管理员回复，支持Markdown格式

- 前台显示控制：开关控制「新留言默认前台可见」，无需手动审核或自动审核

- 系统配置：修改网站标题、邮件配置、版权LOGO/文字/链接、管理员密码

- 邮件通知：新留言提交后，自动发送邮件通知管理员（可配置接收邮箱）

- 邮件日志：查看所有邮件发送记录（成功/失败状态）

- 图片管理：删除留言时自动删除关联上传图片，避免占用空间

### 其他特性

- 数据持久化：留言、配置、图片、日志均持久化存储，重启容器不丢失

- 图片压缩：版权LOGO自动压缩为200×200正方形，优化加载速度

- 响应式设计：适配电脑、手机等不同设备，界面美观简洁

- 错误处理：完善的错误提示（如图片过大、留言为空、密码错误等）

## 📌 技术栈

- 后端：Python + Flask（轻量Web框架）

- 前端：HTML + CSS + JavaScript（原生开发，无需额外依赖）

- 数据存储：JSON文件（无需数据库，部署简单）

- 部署：Docker（容器化部署，跨平台兼容）

- 其他：Markdown渲染、Base64图片处理、SMTP邮件发送、IP归属地查询

## 🚀 部署步骤（Docker方式，推荐）

### 前提条件

- 已安装Docker（版本≥18.09）

- 服务器开放对应端口（默认20000端口，可修改）

- 拥有可用于发送邮件的邮箱（如QQ邮箱，需获取授权码）

### 部署命令

```bash
# 1. 创建数据持久化目录（用于存储留言、配置、图片、日志）
mkdir -p ~/lhcguestbook-data/data ~/lhcguestbook-data/static

# 2. 启动Docker容器（映射20000端口，挂载数据目录）
docker run -d --name lhcguestbook -p 20000:5000 \
  -v ~/lhcguestbook-data/data:/app/data \
  -v ~/lhcguestbook-data/static:/app/static \
  lhcguestbook

# 3. 查看容器运行状态（确认无报错）
docker logs lhcguestbook
```

### 访问地址

- 前台页面：http://服务器IP:20000

- 后台管理页面：http://服务器IP:20000/login

## 🔧 初始配置（首次使用）

### 1. 管理员登录

- 默认管理员密码：123456

- 登录后建议立即修改密码（后台「系统配置」→「修改管理员密码」）

### 2. 系统基础配置

进入后台「系统配置」，修改以下核心配置：

- 网站标题：修改为自己的留言板名称（如「我的个人留言板」）

- 邮件配置（可选，需开启邮件通知）：
        

    - SMTP服务器：如QQ邮箱填 smtp.qq.com

    - SMTP端口：465（默认，无需修改）

    - 发送邮箱：自己的QQ邮箱（如 123456@qq.com）

    - 邮箱授权码：QQ邮箱需在「设置→账户」中开启POP3/SMTP，获取授权码

    - 接收通知邮箱：接收新留言通知的邮箱（可与发送邮箱一致）

- 版权配置：上传版权LOGO、设置LOGO链接、版权文字（如「© 2026 个人留言板 版权所有」）

- 前台显示控制：勾选「新留言默认前台可见」则新留言自动显示，不勾选则需管理员手动审核后显示

### 3. 图片上传配置

- 前台最多可上传4张图片，单张图片≤2M

- 上传的图片会自动存储在挂载的 static 目录，删除留言时会自动删除关联图片

## 📖 使用说明

### 前台使用

1. 访问前台页面，填写昵称（可选）、留言内容（必填）

2. 点击「选择图片」可上传最多4张图片，支持预览和删除已选图片

3. 点击表情面板的表情，可快速插入到留言内容中

4. 点击「提交留言」，留言成功后会显示对应提示（根据前台显示控制开关，显示或待审核）

5. 可对已显示的留言进行点赞，查看管理员回复

### 后台使用

1. 登录后台后，可查看所有留言（包括隐藏和显示状态）

2. 留言操作：
        

    - 回复：输入回复内容，点击「提交回复」，回复会显示在前台对应留言下方

    - 显示/隐藏：点击「前台显示」/「前台不显」，切换留言的前台显示状态

    - 修改点赞数：输入新的点赞数，点击「修改点赞」

    - 删除留言：点击「删除留言」，确认后删除留言及关联图片、邮件日志

3. 邮件日志：点击「邮件日志」可查看所有新留言的邮件发送记录，包括成功/失败原因

4. 配置修改：修改系统配置后，点击「保存配置」即可生效（无需重启容器）

## ⚠️ 注意事项

- 数据持久化：请勿删除 ~/lhcguestbook-data 目录，否则会丢失留言、配置、图片等所有数据

- 图片限制：单张图片≤2M，最多4张，超出限制会提示保存失败

- 邮箱配置：若邮件发送失败，检查SMTP服务器、端口、授权码是否正确，服务器是否开放465端口

- 安全提示：管理员密码建议定期修改，避免泄露；禁止上传违规图片

- 重启容器：修改 app.py 或 templates 目录下的文件后，需重启容器生效（docker restart lhcguestbook）

- 前台显示控制开关：仅对开关设置后提交的新留言生效，不影响历史留言状态

## 🔄 常见问题排查

1. 问题：前台上传图片后，留言中不显示图片
       
解决：检查 app.py 中 MAX_IMG_COUNT 是否为4，index.html 中 MAX_IMAGES 是否为4；重启容器后重试

2. 问题：后台「前台显示控制」开关需要保存两次才生效
        
解决：属于浏览器渲染小延迟，不影响功能，第一次保存已实际生效，刷新页面即可显示正确状态

3. 问题：邮件通知不生效
        
解决：检查邮件配置是否正确，授权码是否有效；查看后台「邮件日志」，根据失败原因排查

4. 问题：重启容器后数据丢失
        
解决：确认启动容器时挂载了 data 和 static 目录，命令是否正确（参考部署步骤）

5. 问题：管理员登录失败
       
解决：默认密码为123456；若修改过密码，确认密码输入正确；若忘记密码，可删除 data/config.json 文件，重启容器恢复默认密码
